## 1. Technology Decision

**React 19 + TypeScript + Vite — separate repository (`TagPulse-UI`)**

See [ADR-007](../adr/007-admin-ui-technology.md) for full rationale. Summary: SPA-grade interactivity (drag-and-drop dashboards, rule builder, real-time charts) is a product requirement informed by Azure IoT Central's capabilities.

### Stack

| Layer | Technology | Purpose |
|-------|-----------|---------|
| Framework | React 19 | Component-based UI |
| Language | TypeScript (strict) | Type safety |
| Build | Vite | Fast dev server + production builds |
| Routing | React Router v7 | Client-side navigation |
| Data fetching | TanStack Query | Server state, caching, refetching |
| API client | openapi-typescript-codegen | Auto-generated from FastAPI OpenAPI spec |
| Components | Ant Design 5 | Tables, forms, modals, layout |
| Charts | Recharts | Time-series, KPI, bar/pie charts |
| Dashboards | react-grid-layout | Drag-and-drop tile arrangement |
| Real-time | EventSource (SSE) | Live data via `/integrations/stream` |
| Linting | ESLint + Prettier | Code quality |
| Testing | Vitest + React Testing Library | Unit + component tests |

### No New Backend Dependencies

Backend already provides everything the SPA needs:
- REST API with OpenAPI spec (auto-generated by FastAPI)
- SSE endpoint for real-time events
- CORS middleware (to be added)

## 3. Pages

### Overview Dashboard (`/`)
- **KPI tiles:** Total devices, reads today, open alerts, anomalies. Auto-refresh via TanStack Query `refetchInterval`.
- **Drag-and-drop layout:** react-grid-layout allows users to rearrange tiles.
- **Recent alerts:** Last 5 alerts with severity badge.
- **Device health grid:** Mini cards per device (online/offline).
- **Live event counter:** SSE connection counting tag reads in real-time.

### Device Management (`/devices`)
- **Fleet table:** Ant Design Table with sorting, filtering, pagination. Columns: name, type, status, connection_state, last_seen.
- **Device detail (`/devices/:id`):** Tabs — Overview (config, metadata), Telemetry (recent reads chart), Health (reads/hour, anomaly flag).
- **Register (`/devices/register`):** Form with name, type, metadata JSON editor.
- **Decommission:** Confirmation modal on detail page.

### Telemetry Dashboard (`/telemetry`)
- **Multi-device chart:** Recharts line chart — reads/min per device over last 24 hours.
- **Live feed:** SSE-powered event stream with smooth chart updates.
- **Device selector:** Dropdown to focus on specific devices.
- **Time range picker:** Last hour / 6h / 24h / 7d / custom.

### Data Explorer (`/telemetry/explore`) — IoT Central pattern (G5)
- **Query builder form:** Device picker, tag ID filter, time range, signal strength range.
- **Results table:** Paginated tag reads.
- **Chart view:** Toggle between table and Recharts visualization.
- **Export:** Download results as CSV (client-side).

### Telemetry Models (`/telemetry-models`) — G6
- **List:** Table with device_type, metric count.
- **Create:** Modal form — device type + dynamic metric rows (name, unit, min, max).
- **Delete:** Confirmation modal.

### Rules & Alerts (`/rules`)
- **Rules table:** Name, condition_type, action_type, enabled toggle, edit/delete actions.
- **Rule editor (`/rules/new`, `/rules/:id/edit`):** Step form — condition config → action config → scope → review.
- **Alert history (`/alerts`):** Table with time, rule, device, severity, status. Filter by status. Acknowledge button.

### Integrations (`/integrations`)
- **Targets table:** Name, type, events, enabled, health_status badge, last_triggered.
- **Create/edit:** Modal form with type-specific config (URL for webhook, events for SSE, schedule for export).
- **Delivery log (`/integrations/:id/deliveries`):** Table with status, attempts, response_code, error.

### Usage & Billing (`/admin/usage`) 
- **Usage chart:** Recharts bar chart — daily usage per dimension.
- **Quota progress:** Progress bars showing current vs. limit per dimension.
- **Summary table:** Aggregated totals per billing period.

## 4. API Client Generation

```bash
# In TagPulse-UI repo:
npm run generate-api

# Runs:
npx openapi-typescript-codegen \
  --input http://localhost:8000/openapi.json \
  --output src/api/generated \
  --client fetch
```

This creates typed TypeScript functions for every API endpoint. No manual fetch calls.

## 5. Authentication

For v1 (before Sprint 12 user auth):
- User enters tenant ID on login page
- Stored in React context (not localStorage — security)
- Sent as `X-Tenant-ID` header on every API request via fetch interceptor

After Sprint 12:
- Login page accepts email + API key
- Backend validates, returns tenant_id + role
- Role-based UI: hide admin pages for viewer role

## 8. Deployment

### Dockerfile

```dockerfile
# TagPulse-UI/Dockerfile
FROM node:20-slim AS build
WORKDIR /app
COPY package*.json .
RUN npm ci
COPY . .
RUN npm run build

FROM nginx:alpine
RUN adduser -D -H appuser
COPY --from=build /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/nginx.conf
EXPOSE 80
HEALTHCHECK --interval=30s --timeout=3s --retries=3 \
  CMD wget -qO- http://localhost/health-ui || exit 1
USER appuser
CMD ["nginx", "-g", "daemon off;"]
```

### nginx.conf

```nginx
worker_processes auto;
events { worker_connections 1024; }

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;
    sendfile on;
    gzip on;
    gzip_types text/css application/javascript application/json;

    server {
        listen 80;
        root /usr/share/nginx/html;
        index index.html;

        # UI health check
        location = /health-ui {
            return 200 '{"status":"ok"}';
            add_header Content-Type application/json;
        }

        # SPA fallback — all UI routes serve index.html
        location / {
            try_files $uri $uri/ /index.html;
        }

        # Proxy API calls to backend service
        location /tag-reads       { proxy_pass http://api:8000; }
        location /device-registry { proxy_pass http://api:8000; }
        location /rules           { proxy_pass http://api:8000; }
        location /alerts          { proxy_pass http://api:8000; }
        location /integrations    { proxy_pass http://api:8000; }
        location /analytics       { proxy_pass http://api:8000; }
        location /admin           { proxy_pass http://api:8000; }
        location /telemetry-models { proxy_pass http://api:8000; }
        location /telemetry       { proxy_pass http://api:8000; }
        location /device-health   { proxy_pass http://api:8000; }
        location /health          { proxy_pass http://api:8000; }
        location /openapi.json    { proxy_pass http://api:8000; }
        location /docs            { proxy_pass http://api:8000; }

        # SSE — disable buffering for streaming
        location /integrations/stream {
            proxy_pass http://api:8000;
            proxy_http_version 1.1;
            proxy_set_header Connection "";
            proxy_buffering off;
            proxy_cache off;
        }
    }
}
```

### Deployment Options

| Option | Frontend | API | Best for |
|--------|---------|-----|----------|
| Docker Compose | `ui` container (nginx) | `api` container | Local dev, small deployments |
| CDN + container | Cloudflare Pages / Vercel / S3 | Docker container | Production at scale |
| Kubernetes | Deployment + Service | Deployment + Service | Enterprise |
